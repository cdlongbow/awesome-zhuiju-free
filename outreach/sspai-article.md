## 正文

---

如果你经常收藏追剧站、TVBox 配置或网盘搜索入口，真正麻烦的往往不是"找不到资源"，而是**"收藏了之后不知道今天还能不能用"**。

为了解决这个问题，我整理了一个开源项目：[Awesome Zhuiju Free](https://github.com/laoma2053/awesome-zhuiju-free)。它不是普通导航站，而是一份放在 GitHub 上公开维护的免费无广告追剧资源清单：资源数据公开、可提交 Issue 参与维护，并且每天自动检测可用性。

浏览器收藏夹里存十几个站，每次真想看剧还要挨个打开试一遍，这件事其实很低效。免费资源本来就不稳定，域名变化、访问限制、服务异常都很常见，与其指望某一个入口永远可用，不如维护一份会持续更新、会标记状态的清单。

---

### 解决思路

与其找一个"永远可用"的网站，不如做一个**能告诉你哪些资源当前可用的开源清单**。

Awesome Zhuiju Free 的核心入口是 GitHub 仓库：  
https://github.com/laoma2053/awesome-zhuiju-free

它的价值主要有三点：

- **开源透明**：资源数据、检测结果、贡献方式都在仓库里公开。
- **人工精选**：只收录免费、无广告体验更好的资源，有明显广告或需付费的站点不会进入。
- **每日检测**：GitHub Actions 每天自动检查资源可访问状态，并把结果展示在 README 和站点页面里。

普通导航站的问题是"收录之后就不管了"。这个项目更像一份活的清单：资源会变化，状态也会跟着更新。

---

### 它是怎么工作的

每天北京时间 09:00，GitHub Actions 自动对清单里的每个资源发请求，根据响应判断状态：

- 🟢 可访问：正常响应
- 🟡 访问受限：返回 401/403 等（站点存在但需要特定条件访问）
- 🔴 无法访问：超时或连接失败
- ⚪ 未检测：开源项目类，不做自动检测

状态直接显示在 README 的资源表格里，也存在 `reports/availability.json`。

不用自己一个个试，打开 GitHub README 就能先看到哪些资源今天更可能可用。需要更直接的浏览体验，也可以访问项目站点：  
https://zhuiju.me

需要说明的是，可用性检测只代表 GitHub Actions 节点在检测时的网络结果，并不等于对第三方资源做永久可用保证。

### 收录了什么

目前项目收录 **73 个资源**，分为以下类别：

| 分类 | 数量 | 说明 |
|---|---|---|
| 在线影视 | 25 | 直接浏览器打开查看 |
| 影视APP | 3 | 手机 / 电视端 App |
| 网盘资源搜索 | 4 | 查找网盘资源入口 |
| 磁力 & BT | 9 | 磁力与 BT 搜索相关资源 |
| 字幕资源 | 3 | 字幕站、字幕组与字幕工具 |
| TVBox / 影视仓空壳 | 2 | TVBox / 影视仓客户端相关资源 |
| TVBox / 影视仓配置地址 | 16 | 配合 TVBox / 影视仓 App 使用 |
| 订阅源 | 1 | IPTV、广播与其他订阅源 |

每个资源都有评分维度（内容量 / 速度 / 界面干净程度 / 稳定性 / 易用性）和风险标注（版权 / 安全 / 隐私 / 支付）。这些信息都放在仓库的结构化数据里，方便维护者持续更新，也方便社区检查。

这也是我更推荐把 GitHub 仓库作为主入口的原因：网站适合快速浏览，GitHub 仓库更适合查看完整数据、检测逻辑、更新记录和参与维护。

### 参与方式

项目完全开源，数据存在 JSON 文件里，有 Schema 验证。

如果你发现某个资源失效，或想推荐一个好用的免费无广告资源：

1. 在 GitHub 提 Issue，选对应模板填写
2. 维护者审核资源是否符合免费、无广告等收录标准
3. 通过后自动写入数据，并同步更新 README 和状态页面

对普通用户来说，不需要懂 Git 或编程；对开发者来说，仓库里的 `resources/resources.json`、Schema 和 GitHub Actions 工作流也都可以直接查看。
